# Radio 单选框组件测试用例

## 组件信息

- **组件名称**: nv-radio
- **组件路径**: `src/components/radio`
- **测试优先级**: P0 (高优先级)
- **组件描述**: 在一组备选项中进行单选

## 1. 功能测试

### 1.1 基础渲染

- **测试ID**: `radio-basic-render`
- **测试步骤**:
  1. 渲染一个基础单选框
  2. 验证单选框正常显示
  3. 验证标签文本正确显示
- **预期结果**: 单选框和标签正常渲染

### 1.2 选中状态

- **测试ID**: `radio-select`
- **测试步骤**:
  1. 点击未选中的单选框
  2. 验证单选框变为选中状态
  3. 尝试再次点击
  4. 验证保持选中状态（单选框一旦选中不能直接取消）
- **预期结果**: 点击后选中，不能直接取消选中

### 1.3 互斥选择

- **测试ID**: `radio-exclusive`
- **测试步骤**:
  1. 创建多个同名单选框
  2. 选中第一个
  3. 验证第一个被选中
  4. 选中第二个
  5. 验证第二个被选中，第一个自动取消
- **预期结果**: 同组内只能有一个被选中

## 2. 属性测试

- **测试ID**: `radio-checked`
- **测试步骤**:
  1. 设置 checked 属性为 true
  2. 验证单选框显示选中状态
- **预期结果**: checked 属性控制选中状态

### 2.2 disabled 属性 - 禁用状态

- **测试ID**: `radio-disabled`
- **测试步骤**:
  1. 设置 disabled 属性
  2. 验证单选框显示禁用样式
  3. 尝试点击单选框
  4. 验证状态不改变
- **预期结果**: 禁用状态下无法操作

### 2.3 label 属性 - 标签文本

- **测试ID**: `radio-label`
- **测试步骤**:

### 2.6 size 属性 - 尺寸

## 5. 交互测试

### 5.1 鼠标点击

- **测试ID**: `radio-mouse-click`
- **测试步骤**:
  1. 使用鼠标点击单选框
  2. 验证被选中

- **预期结果**: 鼠标点击正常工作

### 5.2 键盘导航

- **测试ID**: `radio-keyboard-navigation`
- **测试步骤**:
  1. 使用 Tab 键聚焦第一个单选框
  2. 按方向键（上/下/左/右）
  3. 验证焦点和选中状态移动到相应单选框
- **预期结果**: 方向键导航正常

### 5.3 Space 键选中

- **测试ID**: `radio-keyboard-space`
- **测试步骤**:
  1. 使用 Tab 键聚焦单选框
  2. 按 Space 键
  3. 验证被选中
- **预期结果**: Space 键选中单选框

### 5.4 Tab 键导航

- **测试ID**: `radio-tab-navigation`
- **测试步骤**:
  1. 聚焦第一个单选框组
  2. 按 Tab 键
  3. 验证焦点移到下一个表单元素（跳过同组其他单选框）
- **预期结果**: Tab 键跳过同组单选框
